# Value Proposition and Design Principles

## Value proposition

**For** home theater audiobook listeners with local or cloud-hosted libraries
**Who need** fast browsing, exact resume, and reliable playback from remote storage
**The product is** a native Android TV audiobook player
**That** indexes remote libraries into a local cache and streams playable assets on demand
**Unlike** video-first TV media players or mobile audiobook apps
**Our product** is optimized for long-form listening, D-pad focus, millisecond progress persistence, and remote source resilience.

## Design principles

### 1. Cache first, never wait on folders
Browsing screens must render from local metadata rather than live remote directory calls.

### 2. The remote is the input model
Every visible action must be reachable and understandable with D-pad focus alone.

### 3. Resume is sacred
Saved playback position is the highest-trust state and must stay visible, durable, and recoverable.

### 4. Source problems need repair paths
SMB, provider auth, network, and permission failures must identify the failed source and offer the right recovery.

### 5. TV layout stays inside the frame
Core text, focus states, and controls must remain legible inside overscan-safe margins.

### 6. Cloud providers are first-class
Dropbox, Google Drive, iCloud Drive, and OneDrive need named setup, authorization, folder browsing, and error treatment.

## Non-goals
- No on-device downloads in Phase 1.
- No third-party metadata scraping in Phase 1.
- No phone-style tab bars, touch gestures, hover-only controls, or live-folder library browsing.
- No generic WebDAV treatment for named first-class cloud providers.
