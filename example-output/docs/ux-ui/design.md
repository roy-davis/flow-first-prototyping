Confirmed brand tokens are not supplied in the PRD. The values below are provisional Android TV prototype defaults.

## Framework

- Framework: Native Android TV, Kotlin, Jetpack Compose for TV
- Component library: androidx.tv:tv-material
- Library version: Not supplied
- Documentation URL: Not supplied
- Styling: Compose theme tokens
- Dark mode: Yes, default for living-room playback
- Custom component overrides: Focus scaling, overscan-safe layout, and cache/source status treatments are required across interactive surfaces.

## Visual tone

Quiet, cinematic, and utility-led for couch listening: large enough for 10-foot reading, restrained enough that playback, focus, source status, and book context remain clear.

## Cross-screen UI decisions

| Area | Decision | Applies to |
| --- | --- | --- |
| Navigation model | Home-led drill-downs; setup and repair are linear branches; Back preserves origin context and playback where relevant. | All screens |
| Primary action hierarchy | One dominant action per state; secondary actions grouped nearby; destructive/repair actions require source-specific reason. | All actions |
| Contextual in-flow actions | Resume, repair, chapter start, speed, seek, and source review appear at the focused object and preserve focus/scroll/playback state. | Library, Book Detail, Now Playing, Sync Progress, Source Repair |
| Focus and input | D-pad only is sufficient; default focus lands on the safest primary action; focus uses scale 1.05-1.1 plus border/glow. | All interactive elements |
| Data freshness | Home, Library, and Book Detail read cached local metadata; setup, sync, repair, and playback asset resolution may call sources. | Data-backed screens |
| Loading states | Preserve cached context during loading; use skeleton/progress only for local cache or sync; never blank a screen while source work runs. | Async screens/actions |
| Empty states | Clear heading, one-line explanation, one primary recovery action; no decorative welcome/marketing copy. | Empty variants |
| Error and recovery | Name the source and failure category; show retry/repair action; hide raw credentials, tokens, URLs, file IDs, and stack traces. | Error variants |
| Offline/degraded states | Cached content remains visible when safe; source-dependent actions are disabled or route to repair. | Network-dependent screens |
| Status indicators | Progress means book or sync progress; warning means partial/degraded; success means source or sync is usable; focus is never color-only. | Status-heavy screens |
| Provider/source identity | SMB and first-class providers are named directly; WebDAV is labelled advanced/custom. | Setup, sync, repair, playback |
| Data density | Use TV-scale grids and lists; card/title text can wrap to two lines; dense lists scroll inside stable regions. | Lists, grids, detail |
| Responsive/layout constraints | Android TV landscape first; 48dp minimum safe margin and 64dp preferred; no bottom nav, tiny edge controls, or nested cards. | All screens |
| Copy conventions | Sentence case; verb + object button labels; plain source-specific errors; no generic 'something went wrong'. | All copy |
| Accessibility baseline | AA contrast target; focus visible from 10 feet; reduced motion swaps scale for border/color emphasis. | All screens |

- Base unit: 8dp

## Layout
- Grid: TV-first horizontal shelves and 4 to 6 column poster/card grids depending on viewport width.
- Breakpoints: Android TV landscape first; tablet/mobile are not Phase 1 priorities.
- Container max width: Full viewport with overscan-safe content area.
- Page padding mobile: N/A.
- Page padding desktop: 64dp default; 96dp on large displays when content allows.

## Shape
- Border radius default: 16dp.
- Border radius scale: 8dp small controls, 16dp cards, 24dp hero/progress panels.
- Elevation style: Tonal layering first; focus uses scale, border, glow, and brighter surface rather than deep shadows.

## Navigation
- Pattern: Home-led drill-downs, linear setup/repair branches, contextual return paths.
- Sidebar width: None in the prototype.
- Mobile navigation: N/A.
- Active state style: Focused item scales to 1.05x to 1.1x, shifts to brighter surface, and shows clear border/glow.

## Interaction patterns

### Loading
- Approach: Cached content stays visible; skeletons only for local cache reads; progress panels for sync; inline loading for buttons.
- Button loading: Keep focus on pressed action, show busy label, prevent duplicate activation.

### Empty states
- Structure: Heading, one-line explanation, one primary recovery action.
- Tone: Direct and calm.

### Error states
- Inline field: Specific field-level message below the failed field.
- Form level: Panel-level explanation with retry or repair action.
- Page level: Source-specific explanation and focused recovery CTA.

### Success feedback
- Style: Short status text or toast plus updated state.
- Toast duration: 4 seconds unless playback state changes immediately.

### Modals
- Sizes: Use only for short confirmation/settings; prefer full-screen TV panels for setup.
- Overlay: Dark scrim, focus trapped.
- Dismiss on overlay click: No; D-pad remote requires explicit Cancel/Back.

### Toasts
- Position: Bottom center within safe area.
- Duration: 4 seconds.

### Confirmation (destructive)
- Style: Focused dialog with destructive action clearly colored.
- Require typing: No for Phase 1 source archive/remove actions unless account-level data is deleted.

## Tone and voice
- Label casing: Sentence case.
- Button copy: Specific verb + object where possible, e.g. "Connect source", "Resume book", "Retry sync".
- Error tone: Plain, specific, recovery-oriented.
- Help text: Short, operational, not tutorial-like.

## Icons
- Set: Android/Material-style TV-compatible icons.
- Sizes: 24dp small, 32dp normal, 40dp empty state.
- Stroke weight: Consistent medium weight.

## Accessibility
- WCAG level: AA target.
- Text contrast: 4.5:1 minimum for body text.
- UI contrast: Focus indicators visible from 10 feet.
- Focus style: Border/glow plus scale, not color alone.
- Reduced motion: Scale effects reduce to color/border changes.

## Notes
- Do not rely on hover, touch gestures, tiny controls, edge-aligned actions, phone tab bars, decorative hero sections, or generic file-manager density.
- Book artwork may appear, but browsing must work when artwork is missing.
