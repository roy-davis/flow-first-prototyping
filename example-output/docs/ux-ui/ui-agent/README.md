## Product and Design Context

This is a native Android TV audiobook player for local SMB shares and first-class cloud providers. The prototype must feel like a fast, remote-native media utility for long-form listening: dark mode, large TV-scale type, clear D-pad focus, overscan-safe spacing, cached-library browsing, exact progress, and source-specific repair. The interface should avoid phone navigation, decorative marketing sections, generic file-manager density, and any control that depends on touch or hover.

## Generation Rules

- Target Android TV, not mobile.
- Use D-pad focus as a first-class state on every interactive element.
- Keep core text and controls at least 48dp from viewport edges; prefer 64dp.
- Home, Library, and Book Detail read from local metadata cache.
- Remote SMB/cloud calls happen only during setup, sync, repair, and playback asset resolution.
- Render critical states as separate frames/artboards when possible.
- Do not expose raw credentials, tokens, provider file IDs, secret URLs, or stack traces.
- Do not collapse first-class cloud providers into generic WebDAV.
- Do not render instruction fields as visible UI text. Literal visible text comes only from CONTENT, COPY REQUIREMENTS, or explicit button/label/message entries.

## Shared Navigation

App launch -> Home. Resume -> Now Playing. Browse -> Library -> Book Detail -> Now Playing. Add source -> Sources. SMB setup -> SMB Setup -> Folder Picker -> Sync Progress -> Discovered Books -> Library. Cloud setup -> Provider Authorization -> Folder Picker -> Sync Progress -> Discovered Books -> Library. Playback/source failure -> Source Repair -> repair branch -> return to origin.

## Acceptance

- All primary and critical-state screens render in dark TV style.
- D-pad focus is visible and distinct on every actionable item.
- Empty, loading, error, permission, partial sync, buffering, and repair states are visually distinct.
- Long titles and dense data do not overlap primary actions.
- Source setup clearly distinguishes SMB, first-class cloud providers, and advanced WebDAV.
