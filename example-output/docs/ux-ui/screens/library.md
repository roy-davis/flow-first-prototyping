### Design intent

- Primary purpose: Browse curated listening-library audiobook metadata quickly from the couch.
- Primary actors: Leo the Living Room Listener
- Primary job: Find and select a book without live remote folder traversal.
- Principle served: Cache first, never wait on folders.
- Entry points: Home Browse library, Sync Progress View library, empty current book, cached library from repair.
- Success outcome: Leo selects a book or understands why the library is empty/stale.
- Content priority: Primary: book grid with focus, title, author, progress. Secondary: source status and Add source or review discovered books/Repair actions.
- Critical variants: loading_cache, loaded_grid, empty_library, stale_source
- User state of mind: Focused on find and select a book without live remote folder traversal.
- Usage context: Android TV, 10-foot viewing distance, D-pad remote, home network/cloud source variability, overscan-safe layout.
- Data range: Zero, typical, maximum, long text, missing artwork/metadata, and source-permission variants must render without overlap.
- Anti-goals: Do not expose raw credentials, OAuth tokens, provider file IDs, secret URLs, stack traces, live folder traversal on cached browsing screens, phone navigation, or hover/touch-only controls.
- Fidelity target: Mid-fi
- Exit paths: Book Detail, Sources, Source Repair, Home.

### Screen generation prompt

Design an Android TV Library screen for Leo to browse curated listening-library audiobook metadata without live remote folder traversal. Use a TV-scale grid or shelves with visible D-pad focus, progress, source badges, and artwork fallbacks. Render loading-cache, loaded-grid, empty-library, and stale-source states, keeping Add source or review discovered books and Repair source available only when relevant.
