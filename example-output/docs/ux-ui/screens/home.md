### Design intent

- Primary purpose: Give the listener the fastest path back into the current audiobook.
- Primary actors: Leo the Living Room Listener
- Primary job: Resume the current book or move to library/source setup.
- Principle served: Resume is sacred.
- Entry points: App launch, Android TV dashboard return, Back from Now Playing.
- Success outcome: Current book resumes at the saved millisecond or source recovery is clear.
- Content priority: Primary: current book, progress, Resume. Secondary: Browse library, source health. Tertiary: Add source.
- Critical variants: resume_available, no_source, source_unavailable
- User state of mind: Focused on resume the current book or move to library/source setup.
- Usage context: Android TV, 10-foot viewing distance, D-pad remote, home network/cloud source variability, overscan-safe layout.
- Data range: Zero, typical, maximum, long text, missing artwork/metadata, and source-permission variants must render without overlap.
- Anti-goals: Do not expose raw credentials, OAuth tokens, provider file IDs, secret URLs, stack traces, live folder traversal on cached browsing screens, phone navigation, or hover/touch-only controls.
- Fidelity target: Mid-fi
- Exit paths: Now Playing, Library, Sources, Source Repair.

### Screen generation prompt

> Target tool: Other

Design an Android TV Home screen for Leo, a couch listener who wants to resume a long audiobook with one remote action. Prioritize cached current book metadata, saved chapter position, source health, and a large focused Resume action inside overscan-safe margins, with secondary Browse library and Add source or Repair source paths. Render resume-available, no-source, and source-unavailable states distinctly; source problems must keep cached context visible and explain recovery without exposing raw provider details.

- [ ] Focus, disabled, loading, success, and error feedback are visually defined for Resume.
