### Design intent

- Primary purpose: Let the source owner choose the correct storage type.

- User state of mind: Focused on pick smb, a first-class cloud provider, or advanced webdav.

### Screen generation prompt

> Target tool: Other

Design an Android TV Sources screen for Sam and Clara to choose where their audiobooks live. Make SMB and Dropbox, Google Drive, iCloud Drive, and OneDrive first-class, D-pad-focusable choices, with WebDAV separated as an advanced custom option. Group choices so the decision is easy from 10 feet, show connected/unavailable statuses where relevant, and avoid generic protocol language for the first-class cloud providers.

### Acceptance criteria

- [ ] A source owner can choose SMB or a named cloud provider without protocol jargon.
- [ ] WebDAV is visibly advanced and not the default path for first-class providers.
- [ ] Focus movement between source choices is obvious and predictable.
- [ ] No ungrouped decision point shows more than four visible choices.
- [ ] Long provider descriptions do not overflow tiles.
- [ ] Focus, disabled, loading, success, and error feedback are visually defined for consequential actions.
- [ ] All content uses realistic mock data and no placeholder text.
