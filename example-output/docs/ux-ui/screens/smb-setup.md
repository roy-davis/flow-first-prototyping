### Design intent

- Primary purpose: Collect and validate SMB source details before folder selection.
- Primary actors: Sam the Server Steward
- Primary job: Connect to a local Samba/Windows share securely.
- Principle served: Source problems need repair paths.
- Entry points: Sources Select SMB, Source Repair Edit SMB credentials.
- Success outcome: The SMB source validates and the user can choose a folder.
- Content priority: Primary: host/share fields and Test connection. Secondary: credential mode and error detail.
- Critical variants: credentials_form, testing_connection, connection_error, anonymous_login
- User state of mind: Focused on connect to a local samba/windows share securely.
- Usage context: Android TV, 10-foot viewing distance, D-pad remote, home network/cloud source variability, overscan-safe layout.
- Data range: Zero, typical, maximum, long text, missing artwork/metadata, and source-permission variants must render without overlap.
- Anti-goals: Do not expose raw credentials, OAuth tokens, provider file IDs, secret URLs, stack traces, live folder traversal on cached browsing screens, phone navigation, or hover/touch-only controls.
- Fidelity target: Mid-fi
- Exit paths: Folder Picker, Source Repair, Sources.

### Screen generation prompt

> Target tool: Other

Design an Android TV SMB Setup form for Sam to validate a local network audiobook source. Keep text entry grouped and readable from 10 feet, show anonymous and username/password modes, and make Test connection the primary action. Render credentials, testing, connection-error, and anonymous states with precise error placement for host, share, credentials, and permissions.
