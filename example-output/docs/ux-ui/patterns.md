# Pattern Library

> File: docs/ux-ui/patterns.md
> Behavioural patterns - intent and constraints, not implementation.
> Generated after screen specs exist; refine after visual generation feedback.

## Patterns

### D-pad focus ladder
**What it is:** Every screen has one safe default focus and predictable movement through primary, secondary, then contextual actions.
**When to use:** All Android TV screens and dialogs.
**When not to use:** Never omit focus for non-interactive status-only content; status can be read but not focused.
**Must achieve:** A remote user can operate the screen without hover, touch, or guessing where focus moved.
**Anti-pattern:** Multiple equally dominant controls or focus rings visible at the same time.

---

### Cached context before remote work
**What it is:** Cached book/source/progress context remains visible while remote validation, sync, repair, or playback resolution happens.
**When to use:** Home, Library, Book Detail, Now Playing, and Source Repair.
**When not to use:** First-time setup screens before a source exists.
**Must achieve:** The user never loses context because a network/provider operation is slow.
**Anti-pattern:** Blank loading screens that hide the current book or selected source.

---

### Source repair ladder
**What it is:** Recovery starts with retry, then branches to credential/auth/folder repair only when the diagnosis requires it.
**When to use:** SMB, cloud auth, sync, and playback source failures.
**When not to use:** Non-source local cache errors.
**Must achieve:** Users understand which source failed and the smallest safe recovery action.
**Anti-pattern:** Generic "something went wrong" errors or immediate full setup restart.

---

### Playback position preservation
**What it is:** Repair, Back, dashboard exit, speed, seek, and errors preserve the current playback position and return target.
**When to use:** Now Playing and any flow entered from playback.
**When not to use:** Source setup before any book is selected.
**Must achieve:** The listener trusts that a 40-hour book position will not be lost.
**Anti-pattern:** Returning from repair to Home without a way back to the failed book.

---

### First-class provider choice
**What it is:** Named cloud providers are shown as direct choices; WebDAV is advanced/custom.
**When to use:** Sources, Provider Authorization, Source Repair.
**When not to use:** Generic capability or status summaries where provider names are already known.
**Must achieve:** Cloud users can choose the provider they recognize without protocol knowledge.
**Anti-pattern:** Treating Dropbox, Google Drive, iCloud Drive, or OneDrive as generic WebDAV.

## Anti-patterns

- Live-folder library browsing: breaks the cache-first performance promise.
- Phone navigation on TV: bottom tabs, tiny edge controls, and gesture-only affordances fail the remote model.
- Raw provider diagnostics: tokens, file IDs, URLs, stack traces, and secret paths must not be shown.
- Decorative landing page: the first viewport must be useful app state, not marketing.

## Open questions

- Source management pattern: whether connected sources need a dedicated management surface.
- Large library search/filter pattern: whether Phase 1 needs search beyond browse and progress metadata.
