# Provider API - External storage connector

> File: docs/ux-ui/actors/provider-api.md
> Actor type: Non-human

**Operational summary:** Provider API represents SMB, Dropbox, Google Drive, iCloud Drive, OneDrive, and optional WebDAV endpoints. It lists folders, resolves playable asset references, refreshes authorization when supported, and introduces provider-specific failures the app must explain clearly.

## Identity

| Attribute | Value |
| --- | --- |
| Actor type | Non-human |
| Actor category | API / storage provider / network service |
| Owner | Third party or customer |
| Boundary | External |
| Environment | LAN or cloud |
| Trust level | Partially trusted external |

## Responsibilities

| Responsibility | Detail |
| --- | --- |
| Inputs | Auth credentials/tokens, folder path, file ID/path, range requests |
| Processing | Authorize, list folders/files, return metadata, resolve stream URLs |
| Outputs | Folder listings, file metadata, playable references, provider errors |
| Timing / frequency | Setup, sync, playback start, token refresh |
| User-visible effect | Source setup, sync progress, playback availability, repair states |

## Capabilities

| Capability | Allowed decision/action | Cannot decide/do |
| --- | --- | --- |
| List content | Return provider folder/file hierarchy | Cannot define app metadata model |
| Resolve playback | Provide SMB URI or provider download URL | Cannot guarantee home network stability |
| Refresh auth | Renew tokens if supported | Cannot recover revoked access without user action |

## Triggers

| Trigger | Source | Result |
| --- | --- | --- |
| User authorizes source | Provider setup | Token/session created |

| Player starts track | Playback engine | Playable asset resolved |
| Token expires | Provider | Repair/reauth required |

## States

| State | Meaning | User-visible behaviour |
| --- | --- | --- |
| Available | Provider responds and auth is valid | Normal browsing/playback |
| Degraded | Rate limit, partial folder access, slow connection | Warning or retry |
| Pending | Auth/device flow waiting | Authorization pending |
| Blocked | Permission revoked or expired | Repair source CTA |
| Unavailable | Network/provider outage | Source unavailable |

## Failure Modes

| Failure | Cause | User-visible impact | Recovery |
| --- | --- | --- | --- |
| SMB host unreachable | Host/network unavailable | Source unavailable | Check network and retry |
| Bad SMB credentials | Credential rejected | Credential error | Edit credentials |
| OAuth expired/revoked | Token invalid | Reauthorization required | Start provider auth |
| Provider lacks ranged download | Capability unsupported | Playback limited warning | Mark unsupported or warn |
| Rate limit | Provider throttling | Sync delayed warning | Retry later |

## Constraints for Flow Design

| Constraint | Implication |
| --- | --- |
| Latency | Never block library browsing on live provider calls |
| Reliability | Separate provider failures from local app failures |
| Permissions | Show provider-specific recovery paths |
| Rate limits / quotas | Avoid aggressive retry loops |
| Privacy / security | Never show raw tokens, passwords, or secret URLs |
| Observability | Expose status through source health, not implementation logs |
