# Sync Worker - Background indexing automation

> File: docs/ux-ui/actors/sync-worker.md
> Actor type: Non-human

**Operational summary:** The Sync Worker crawls connected SMB and cloud sources in the background, interprets folders and supported audio files as books and chapters, and writes metadata into the local database so browsing never depends on live remote folder traversal.

| Attribute | Value |

| Owner | Product |
| Boundary | Internal |
| Environment | Android WorkManager |

## Responsibilities

| Responsibility | Detail |
| --- | --- |
| Inputs | Source configuration, credentials/tokens, provider capabilities, folder path |
| Processing | Crawl folders, read metadata/ID3, calculate IDs, map files to chapters |
| Outputs | Local metadata rows, sync status, partial error records |
| Timing / frequency | Initial source sync, manual retry, scheduled background refresh |
| User-visible effect | Progress screen, library availability, source health |

## States

| --- | --- | --- |

| Pending | Waiting for constraints | Queued status |

| Unavailable | Worker/system failure | Error with retry |

## Failure Modes

| Failure | Cause | User-visible impact | Recovery |
| --- | --- | --- | --- |
| Source unreachable | Network/provider unavailable | Source unavailable warning | Retry or repair source |
| Auth expired | Provider token invalid | Reauthorization required | Open provider repair |
| Unsupported file | File extension/codec unsupported | Partial sync warning | Skip file and show count |
| Large library slow | Many folders/tracks | Long-running sync progress | Keep cached library usable |

## Constraints for Flow Design

| Constraint | Implication |
| --- | --- |
| Latency | UI must show progress without blocking navigation |

| Privacy / security | Do not expose credentials or token details |
| Observability | Source health and last sync should be visible |
