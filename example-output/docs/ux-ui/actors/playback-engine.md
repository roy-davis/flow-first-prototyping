# Playback Engine - Media playback service

> File: docs/ux-ui/actors/playback-engine.md
> Actor type: Non-human

**Operational summary:** The Playback Engine resolves stored playable asset references into streamable media, starts Media3 playback, maintains MediaSession foreground behavior, and persists progress every 5 seconds and on pause, skip, or app termination.

## Identity

| Attribute | Value |
| --- | --- |
| Actor type | Non-human |
| Actor category | System / service |
| Owner | Product |
| Boundary | Internal |
| Environment | Android Media3 ExoPlayer + MediaSessionService |
| Trust level | Trusted internal process |

## Responsibilities

| Responsibility | Detail |
| --- | --- |
| Inputs | Book/chapter metadata, playable asset reference, saved position, playback command |
| Processing | Resolve asset, buffer stream, play audio, update position, expose media controls |
| Outputs | Playback state, progress, errors, media session status |
| Timing / frequency | On playback start and during active playback |
| User-visible effect | Now Playing controls, background audio, progress persistence |

## Capabilities

| Capability | Allowed decision/action | Cannot decide/do |
| --- | --- | --- |
| Start playback | Stream from resolved asset | Cannot play without reachable source |
| Persist progress | Save exact millisecond position | Cannot guarantee external provider availability |
| Handle controls | Pause, seek, speed change | Cannot fix credentials without user action |

## Data Ownership

| Data | Reads | Writes | Retains | Exposes to user |
| --- | --- | --- | --- | --- |
| Playback position | Yes | Yes | Local database | Progress and resume state |
| Playable asset reference | Yes | Resolved only | Local database/provider | Availability status |
| Playback error | Yes | Yes | Session/log state | Recovery message |

## Triggers

| Trigger | Source | Result |
| --- | --- | --- |
| Resume/start book | User action | Resolve and start track |
| Playback active | Timer every 5 seconds | Save progress |
| Pause/skip/exit | User or app lifecycle | Persist immediate position |

## States

| State | Meaning | User-visible behaviour |
| --- | --- | --- |
| Available | Can resolve and stream | Playing or paused controls |
| Degraded | Buffering or slow stream | Buffering state with controls |
| Blocked | Source cannot resolve playable asset | Repair source CTA |
| Unavailable | Playback service failure | Playback error |

## Failure Modes

| Failure | Cause | User-visible impact | Recovery |
| --- | --- | --- | --- |
| Source dropout | Network/provider failure | Buffering then error | Retry or repair source |
| Unsupported file | Codec/format unsupported | Cannot play chapter | Skip or resync warning |
| Progress write failure | Local database unavailable | Resume risk warning | Retry save on next tick |

## Constraints for Flow Design

| Constraint | Implication |
| --- | --- |
| Latency | Start stream within 2000ms under typical 5GHz Wi-Fi |
| Reliability | Buffer through brief network drops |
| Permissions | Needs source access and media session permissions |
| Rate limits / quotas | Avoid repeated asset-resolution loops |
| Privacy / security | Do not expose direct URLs or file IDs |
| Observability | Expose user-safe playback state and recovery |
